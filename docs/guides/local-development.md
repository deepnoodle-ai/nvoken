# Local workflows

The former local development quickstart mixed product evaluation with source
development. Choose the workflow that matches what you are trying to do:

- [Run nvoken locally](run-locally.md) using the official Homebrew binaries and
  public TypeScript package. Start here if you want to try or integrate nvoken.
- [Develop nvoken](developing-nvoken.md) from a source checkout. Start here if
  you intend to change the daemon, SDKs, examples, or documentation.
- [Deploy nvoken](../../deploy/single-daemon/README.md) only when you are ready
  to operate a production installation.

Run and Develop both use the daemon's disposable quickstart automation.
`deploy/local` retains lower-level Compose and configuration assets for tests
and debugging; it is not the first-time path or a production deployment
profile.
